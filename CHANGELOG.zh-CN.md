# 更新日志

## 0.5.0 - 2026-06-09

CodeSeeX 0.5.0 是 Rust/Tauri 架构版本，重点是更快的启动速度、更稳定的桌面常驻能力、Codex 工具兼容、干净的运行数据、更强的 Web Search、新增视觉模块，以及更顺滑的桌面安装体验。

### 发布重点

- 桌面运行时切换到 Rust + Tauri 2，性能和常驻稳定性明显提升。
- 强化 Codex 工具兼容，包括原生 Apply Patch、原生 MCP 透传，以及更稳妥的工具归属判断。
- 新增视觉模块，可通过用户配置的 OpenAI 兼容端点进行识图和生图。
- 增强 Web Search，优化网页抽取、结果整理、本地/私有地址保护，并支持系统代理配置。
- 更新 Codex 模型目录，为 `deepseek-v4-pro` 和 `deepseek-v4-flash` 提供 1M context 与 95% 自动压缩阈值。
- 优化桌面 UI、工具卡片、图标、日志、用量展示、设置布局和安装器流程。

### 新增

- 新增 Rust/Tauri 桌面管理器，支持托盘、开机自启、更新检查、日志、用量、余额和设置管理。
- 新增视觉模块配置项：识图请求地址、生图请求地址、模型名称和 API Key。
- 新增 `vision_analyze`，支持 HTTP(S) 图片、`data:image`、`file://`、工作区路径，以及权限允许的本地绝对路径。
- 新增 `vision_analyze` 对 Codex 当前消息 `input_image` 附件的直接处理能力，无需通过 shell 转 base64 或手动复制到工作区。
- 新增原生兼容的 `image_gen` 生图工具暴露，并保留 `vision_generate` 作为兼容入口。
- 新增目录列表、文件读取、工作区搜索、Web Search、Apply Patch、Vision 等工具的独立图标。
- 新增 NSIS EXE 安装器流程，支持安装语言、当前用户/所有用户安装模式，以及旧桌面版本平滑升级。

### 优化

- 优化 Apply Patch schema 和原生透传逻辑，文件修改继续由 Codex 原生工具负责。
- 优化 MCP 处理，Codex 原生 MCP/外部工具会继续暴露给模型，并回到 Codex 执行。
- 优化长会话、压缩历史和工具结果的上下文重放稳定性。
- 优化同一轮托管工具执行逻辑，多个 CodeSeeX 工具可以并发执行，同时保持稳定的 Codex 回放顺序。
- 优化运行日志结构，用户日志更精简，诊断日志默认不落盘，超长工具摘要会在写入前压缩。
- 优化运行态数据边界，完整 Codex 请求正文不进入运行态存储，`data:` 与 `input_image` 载荷会在进入日志或回放状态前脱敏。
- 优化工具描述，改成面向用户的能力说明，不再展示实现细节。
- 更新生成的 Codex TOML 和 model catalog，适配当前 Codex 的目录字段要求，并自动刷新已存在但过期的生成目录。
- 优化 Web Search 和只读工作区工具，执行边界更明确，结果摘要更干净。
- 优化生图结果处理，上游返回的 base64 图片会保存为本地文件，并以可直接展示的 Markdown 返回，不再作为大段内联工具结果返回。
- Codex 标题生成、话题建议等轻量辅助请求会路由到 Flash；完整对话 GPT-5 alias 使用配置的默认回退。
